# 迷你商店經營 🏪

一個可在**電腦瀏覽器**和**手機瀏覽器**上玩的 2.5D 等距商店經營遊戲。
佈置你的店面（貨架、櫃台、牆、門），顧客會自己**走進來逛貨架、到櫃台結帳付錢**，
店面進度會**自動存檔**。純 HTML/CSS/JavaScript，**免安裝、免打包**。

> 🌐 線上遊玩：https://a26222239-sketch.github.io/Simulation-Games/

---

## 怎麼玩

- 下方工具列選 **貨架 / 櫃台 / 牆 / 門 / 清除**，點格子就能佈置店面。
- 顧客會自動從**門**進場 → 走到**貨架**前逛 → 到**櫃台**結帳（入帳）→ 從門離開。
- 上方顯示：資金、店內人數、已服務人數、營業狀態。
- 右上 ☰：暫停/開始營業、存檔/讀檔、匯出/匯入、重設店面。

| 操作 | 電腦 | 手機 |
|------|------|------|
| 佈置 / 清除 | 點格子 | 點格子 |
| 平移 | 拖曳空白處 | 單指拖曳 |
| 縮放 | 滾輪 | 雙指 |

### 本機執行
因為用了 ES Modules，請用本機伺服器開（直接雙擊 index.html 少數瀏覽器會擋）：
```bash
python3 -m http.server 8000   # 然後開 http://localhost:8000
```

## 動畫顧客怎麼運作（重點觀念）

顧客「會走來走去」是**兩件事疊加**，都由程式控制，**不是用 GIF**：

1. **逐格動畫**：走路分解動作。目前用程式畫的小人（腿會擺動）；
   之後可換成**精靈圖**（見下）。
2. **位移 + 尋路**：`js/customer.js` 的狀態機讓顧客在格子間連續移動，
   `js/pathfind.js`（BFS）負責繞過貨架找路。

---

## 換成你的 GPT 立繪 / 顧客動畫 🎨

程式內建「**有圖就用圖、沒圖就用內建圖形**」，丟檔進 `assets/` 即可，**不用改程式**。

### 設施（地板/牆/貨架/櫃台/門）
放 `assets/<名稱>.png`：`floor.png`、`wall.png`、`shelf.png`、`counter.png`、`door.png`。
透明背景，底部寬度 ≈ 一格菱形寬（約 128px）。各設施的外型建議寫在 `js/config.js` 的 `art` 欄位。

### 顧客走路動畫（精靈圖 sprite sheet）
放一張 `assets/customer.png`，排成 **4 列（方向：下/左/右/上）× 4 行（走路分解動作）**，
單格建議 48×64px、透明背景。程式會自動裁切對應方向與動作格播放。
> 小技巧：請 GPT「同一個角色、只改腿部姿勢」生 4 格；只要 2 格交替也夠看。

## 檔案結構

```
index.html        畫面與 UI
css/style.css     樣式（含手機響應式）
js/config.js      規則設定：店面大小、設施、顧客參數、立繪外型建議
js/iso.js         等距座標換算（格子 <-> 螢幕）
js/store.js       商店狀態：店面、編輯、生顧客、營收、存讀檔
js/customer.js    顧客行為（進場→逛→結帳→離場）的狀態機與動畫狀態
js/pathfind.js    BFS 尋路（繞過貨架）
js/render.js      畫面繪製（地板/設施/顧客，含深度遮擋）
js/input.js       滑鼠 / 觸控（平移、縮放、點擊放置）
js/main.js        進入點：組裝場景並啟動
assets/           放你的立繪 / 顧客精靈圖
```
